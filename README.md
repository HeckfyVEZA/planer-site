# My Day Tracker

Статический мини-планер на чистых `HTML`, `CSS` и `JavaScript` без сборки и внешних зависимостей.

## Что изменено

- Монолитный `index.html` разделён на три независимых слоя: `index.html`, `styles.css`, `app.js`.
- Поведение приложения переработано вокруг явных зон ответственности: состояние, storage, рендер, события, date-utils.
- Из динамической разметки убраны небезопасные вставки пользовательского текста через `innerHTML`, где это было оправдано. Теперь пользовательские данные рендерятся через DOM API.
- Убрано дублирование обработчиков за счёт event delegation для списков задач, календаря и списка фраз.
- Сохранена исходная визуальная композиция: сетка, стеклянные карточки, типографика, цвета, адаптивные брейкпоинты.
- Добавлены базовые улучшения доступности: `aria-*`-атрибуты, состояния табов, диалоговая модалка, focus states.
- Добавлены favicon, `meta description`, `theme-color`, а также дополнительные безопасные визуальные улучшения без смены общей стилистики.
- Ускорен рендер календаря и модалки за счёт подготовки карт задач по датам вместо повторных фильтраций на каждый день.

## Структура файлов

- `index.html` — семантическая разметка, точки монтирования UI, доступность, подключение ассетов.
- `styles.css` — все стили интерфейса, токены темы, responsive-поведение, состояния компонентов.
- `app.js` — состояние приложения, чтение и запись в `localStorage`, рендер задач, календаря, заметок и фраз.
- `README.md` — документация по устройству и поддержке.

## Архитектура JavaScript

- `state` хранит задачи, заметки, фразы, активную вкладку, режим календаря и выбранную дату модалки.
- `load*` и `save*` отвечают за работу с `localStorage` и нормализацию данных.
- `render*` функции отвечают только за отображение и не смешиваются с логикой инициализации.
- `handle*` функции инкапсулируют реакции на пользовательские события.
- `create*` функции собирают DOM-узлы для задач, карточек календаря, превью и фраз.

## Данные в localStorage

- `my-day-soft-tracker` — список задач.
- `my-day-soft-notes` — заметки по датам в формате `YYYY-MM-DD`.
- `my-day-soft-phrases` — пользовательские мотивационные фразы.

## Как запускать

1. Открой `index.html` в браузере.
2. Приложение работает полностью локально.
3. Все пользовательские данные сохраняются в `localStorage` текущего браузера.

## Что важно при дальнейшей поддержке

- Если добавляется новый UI-блок, сначала определить, где он живёт: в `state`, `render` или `events`.
- Для пользовательских строк предпочтительно использовать `textContent`, а не HTML-строки.
- Для новых интерактивных списков лучше сохранять event delegation, а не навешивать обработчики на каждый элемент.
- Визуальные токены лучше расширять через `:root`, а не хардкодить цвета в компонентах.
